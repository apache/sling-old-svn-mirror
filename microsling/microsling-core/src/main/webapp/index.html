µsling homepage

Overview

µsling (pronounced microsling) started as "back to basics" rewrite of Sling, which is heavier, using OSGi, and at the time was much harder to understand.

As I write this (early December 2007), µsling is perfectly usable to create scripted JCR-based applications (it runs dev.day.com for example). We have decided to focus on javascript, using it for either server-side or client side templates (see links below), or for handling other HTTP requests besides GET.

The ujax.js client-side javascript library provides a simple javascript "proxy" on the JCR repository. The name ujax should really by written µjax (microjax), but for all filenames and technical names (parameters etc.) with use "ujax" to avoid encoding problems.

Documentation and examples are scarce for now, but the automated tests (both unit and integration tests) are extensive and up to date, if you can read simple java code they might help.
